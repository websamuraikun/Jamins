## Architecture

This is a **Next.js 16 + Prismic CMS** website using the App Router.

### Content Flow

Prismic is the headless CMS. Pages are composed of **slices** — modular content sections. The content model is defined locally in Slice Machine and synced to Prismic.

- `slicemachine.config.json` — repository name and slice library config
- `src/prismicio.ts` — Prismic client setup, route resolvers, and revalidation strategy
- `prismicio-types.d.ts` — auto-generated TypeScript types from Prismic models (do not edit manually)
- `customtypes/` — Prismic custom type definitions (page, settings, navigation)

### Slices

Slices live in `src/slices/`. Each slice is a self-contained directory:
```
SliceName/
├── index.tsx     # React component
├── model.json    # Content model (managed by Slice Machine)
└── mocks.json    # Mock data for Slice Machine preview
```

`src/slices/index.ts` is **auto-generated by Slice Machine** — do not edit manually. When a new slice is created via the Slice Machine UI, it's registered here automatically.

Pages render slices via `<SliceZone slices={page.data.slices} components={components} />`.

### Routing

- `src/app/page.tsx` — home page (`uid: "home"`)
- `src/app/[uid]/page.tsx` — all other pages (dynamic, statically generated via `generateStaticParams`)
- `src/app/api/preview/route.ts` — Prismic preview route
- Route resolvers in `src/prismicio.ts` define the URL structure

### Styling

- **Tailwind CSS 4** via PostCSS (no `tailwind.config.js` — uses CSS-based config)
- Custom design tokens defined in `src/app/globals.css` under `:root` and `@theme`
- Fonts loaded via `next/font/google` in `src/app/layout.tsx` with CSS variables:
  - `--font-display` → DM Serif Display (headings)
  - `--font-body` → DM Sans (body text)
  - `--font-inter` → Inter (Tailwind `font-sans`)
- Reusable CSS classes (`.btn`, `.section-title`, `.section-label`, etc.) defined in `@layer components` in `globals.css`

### Data Fetching

- All data fetching is server-side via async Server Components
- Production: `cache: "force-cache"` with `next: { tags: ["prismic"] }` for on-demand revalidation
- Development: `next: { revalidate: 5 }` (5-second revalidation)
- `NEXT_PUBLIC_PRISMIC_ENVIRONMENT` env var overrides the repository name (used for environment branches)
